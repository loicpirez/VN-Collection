# I18n Completeness Audit — `/Users/loicpirez/VNDB`

Scope: every `.tsx` file under `src/app/` and `src/components/` (275 files). i18n stack: `src/lib/i18n/dictionaries.ts` (FR/EN/JA) consumed via `getDict()` (server) and `useT()` (client). Dictionary structure: 126 top-level sections per locale, identical names/order across all three locales.

## Methodology

1. Confirmed top-level section parity across `fr`/`en`/`ja` blocks — all 126 keys present in all three locales.
2. No `'TODO'` markers and no empty-string leaf values across the 8 937-line dictionary file.
3. Scanned for `aria-label="..."`, `title="..."`, `placeholder="..."`, `alt="..."` literal strings and for hardcoded JSX children. Most surfaces correctly use `t.*` paths.
4. Scanned `throw new Error(...)`, `console.error`, and toast/notification paths for English text that bleeds into user-facing displays.
5. Scanned for `.toLocaleString(...)`, `.toLocaleDateString(...)` etc. to find places that bypass `useLocale()`.
6. Verified every `replace('{placeholder}', ...)` callsite has the matching placeholder in the dict string (spot-checked the most important keys).
7. Searched for English plural-assumption patterns like `${n} result${n>1?'s':''}` — none present (good).

## Notes on what was inspected but found clean

- All `aria-label={...}` attributes (175 occurrences across `src/app/` + `src/components/`) read from `t.*` paths, except I-001.
- `placeholder="..."` literals are either numeric examples (`"1080"`, `"1280"`, `"720"`, `"1999"`, `"2026"`, `"5000"`, `"70"`) or URL examples (`"https://…"`, `"proxy.example.com"`). The dictionary already covers every user-facing placeholder via `t.*.searchPlaceholder` etc.
- `title="..."` attributes — none hardcoded with English UI text; they receive `t.*` values or data fields (VN titles, character names, etc.).
- `alt="..."` attributes — three `alt=""` (`HeroBanner.tsx:322,350`, `VnSourcePicker.tsx:144`) are intentional empty-alt for decorative thumbnails alongside accessible siblings.
- JSDoc / source comments contain English UI words like "Cancel"/"Save" — confirmed they're only comments, never rendered.
- The `console.error` calls across `src/app/**/error.tsx` and similar are dev-only logs (never displayed to the user). `useConfirm()` and `useToast()` error paths route through `(e as Error).message` which carries localised content when the caller wrapped via `readApiError(r, t.common.error)`.
- `formatMinutes` (`src/lib/format.ts`) is the canonical helper. When called with `t.year`, it correctly localizes. The defect list I-014/18/19/20/21/22 is about callers that bypass it; the helper itself is fine.
- Steam page error code detection (I-016) is the only "string-comparison-against-English" bug; the rest of the codebase compares against stable codes (`'auto'`, `'manual'`, `'planning'`, `'completed'`, etc. — those are domain enums, not user-visible text).

## Recommended fix priorities

1. **HIGH I-001, I-002, I-003, I-004, I-005, I-006**: short user-visible English strings missing from the dictionary. Each is a 1-line dict addition + 1-line code change. Quick wins.
2. **HIGH I-007**: classification warnings persisted in the DB. Should migrate to opaque enum-style keys (e.g. `bonus_only`, `related_music_title`) and resolve to a localised label at render time in `<StockPanel>`. Migration path: store the enum key going forward; render either the new key via dict or fall back to the legacy English string for old rows.
3. **HIGH I-008, I-027**: `'Online stock'` sentinel — same pattern as I-007, fix together.
4. **HIGH I-009, I-010, I-011**: standardise on `readApiError(r, t.common.error)` everywhere (already used in 50+ places), kill the four `\`HTTP ${r.status}\`` throws.
5. **HIGH I-012**: pass `locale` to `toLocaleString()` in AliceNetKobeClient (1-character fix).
6. **MEDIUM I-013, I-014, I-015, I-016**: `upcoming/page.tsx` TBA bucket; Steam page `fmt` helper + raw `l.source` chip + error string comparison.
7. **MEDIUM I-017–I-022**: factor every local `fmt`/`fmtMin` to the shared `formatMinutes`. Recommend tightening `formatMinutes` signature (see I-025) so the failure mode is loud, not silent.
